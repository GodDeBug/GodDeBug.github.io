idea数据库配置

##关于第一次配置数据库
需要先去载入jdbc程序驱动，及mysql的jar包详见：https://blog.csdn.net/Com_ma/article/details/80505179（mysql的jar包不同于mysql需要另外下载）
配置文件导入成功后，在project data sources中选择mysql
在左侧的driver filer中导入刚刚下载的mysql程序驱动完成
再在database中创建数据库（其中所要用的database必须存在，否则失败）
##mysql常用jar包
连接Mysql数据库：

　　常用的连接池有两种

DBCP连接池
C3P0连接池

Apache的commons组件 – DBCP连接池:

commons-dbutils-1.4.jar　　封装并简化了JDBC

commons-dbcp-1.4.jar　　 提供的数据库连接池组件,命名为DBCP

commons-pool-1.3.jar　　　DBCP连接池依赖该jar包

mysql-connctor-java-5.1.28-bin.jar　　Mysql的JDBC驱动包，用JDBC连接数据库必须使用该jar包

C3P0连接池:
c3p0-0.9.1.2.jar　　　　　 使用c3p0连接池所使用的jar包

##Intellij IDEA载入JDBC驱动，连接Mysql
1、https://blog.csdn.net/qq_41342884/article/details/89149995?depth_1-utm_source=distribute.pc_relevant.none-task&utm_source=distribute.pc_relevant.none-task（图文版）

2、
第一步，你得去官网下载https://dev.mysql.com/downloads/connector/j/

第二步，解压压缩包，随便放哪都可以

第三步，打开你的idea工程，按ctrl+alt+shift+s（此为打开Project Structure），Modules —>>选中–>>点击右侧的＋号——>选第一个jars of directories，找到你刚刚解压缩的位置，选中文件mysql-connector-java-5.1.42-bin.jar——>点确定

第四步，查看你的Extermal libraries

##数据库创建失败（Server returns invalid timezone. Go to ‘Advanced’ tab and set ‘serverTimezon）
主要是mysql时区错误，需要在mysql下修改时区。
展示时区： show variables like’%time_zone’; 显示 SYSTEM 就是没有设置时区啦
修改时区：SET GLOBAL time_zone = ‘+8:00’;
SET time_zone = ‘+8:00’;
刷新：FLUSH PRIVILEGES;
修改时区操作链接：https://blog.csdn.net/liuqiker/article/details/102455077
